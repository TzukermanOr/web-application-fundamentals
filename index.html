HTML

מבנה דף HTML

HTML - שפת תגיות שמגדירה את המבנה והתוכן של דף ווב. כל דף HTML בנוי מתגיות מקוננות שיוצרות עץ היררכי.

📄

DOCTYPE

הצהרה לדפדפן שזהו מסמך HTML5. תמיד בשורה הראשונה.

🧠

head

מטא-דאטה: כותרת הטאב, קידוד, CSS, ומידע שלא נראה בדף עצמו.

👁️

body

כל התוכן הנראה למשתמש: טקסט, תמונות, קישורים, טפסים.

השלד הבסיסי

index.html 
<!-- הצהרה שזה מסמך HTML5 -->
<!DOCTYPE html>
<html lang="he" dir="rtl">

<head>
    <!-- קידוד תווים - תמיד UTF-8 -->
    <meta charset="UTF-8">

    <!-- התאמה למובייל -->
    <meta name="viewport" content="width=device-width, initial-scale=1.0">

    <!-- כותרת שמופיעה בטאב -->
    <title>שם האתר</title>

    <!-- קישור לקובץ CSS חיצוני -->
    <link rel="stylesheet" href="style.css">
</head>

<body>
    <header>
        <nav>תפריט ניווט</nav>
    </header>

    <main>
        <h1>כותרת ראשית</h1>
        <p>תוכן הדף</p>
    </main>

    <footer>
        <p>כותרת תחתונה</p>
    </footer>
</body>

</html>
💡 טיפ

תגיות סמנטיות כמו <header>, <main>, <footer> עדיפות על <div> גנרי - הן עוזרות לקוראי מסך, ל-SEO, ולקריאות הקוד.

CSS

יסודות CSS

CSS - השפה שמעצבת את מה ש-HTML בנה. צבעים, גדלים, מיקום, אנימציות - הכל כאן.

3 דרכים לשלב CSS

css-methods.html 
<!-- 1. Inline - ישירות על האלמנט (לא מומלץ) -->
<p style="color: red; font-size: 18px;">טקסט אדום</p>

<!-- 2. Internal - בתוך תגית style ב-head -->
<style>
    p { color: red; }
</style>

<!-- 3. External - קובץ חיצוני (מומלץ!) -->
<link rel="stylesheet" href="style.css">

סלקטורים - איך "מכוונים" לאלמנטים

p כל תגיות <p>
.my-class כל אלמנט עם class="my-class"
#my-id האלמנט הספציפי עם id="my-id"
div p כל p שנמצא בתוך div (צאצא)
div > p רק p שהוא ילד ישיר של div
a:hover קישור כשעוברים עליו עם העכבר
input:focus שדה קלט שמוקד (נלחץ עליו)
li:first-child הפריט הראשון ברשימה

CSS

Box Model

כל אלמנט ב-HTML הוא קופסה. ה-Box Model מגדיר 4 שכבות סביב התוכן.

הדגמה חיה

content
box-model.css 
.element {
    /* Content - התוכן עצמו */
    width: 200px;
    height: 100px;

    /* Padding - ריווח פנימי (בין התוכן לגבול) */
    padding: 20px;            /* כל הצדדים */
    padding: 10px 20px;        /* למעלה/למטה | ימין/שמאל */
    padding: 10px 20px 30px 40px; /* top right bottom left */

    /* Border - הגבול */
    border: 2px solid #333;

    /* Margin - ריווח חיצוני (בין האלמנט לשכנים) */
    margin: 16px;

    /* חשוב! box-sizing שולט איך מחושב הרוחב */
    box-sizing: border-box;   /* width כולל padding + border */
}
🔑 קריטי

תמיד הגדירו box-sizing: border-box על כל האלמנטים (דרך ה-reset). בלי זה, padding ו-border מתווספים מעבר ל-width ויוצרים חישובים מבלבלים.

💡 מתי להשתמש במה?

Flexbox - כשהפריסה היא בכיוון אחד: שורת כפתורים, תפריט, מרכוז.
Grid - כשצריך שליטה בשני ממדים: דף שלם, גלריה, דשבורד.
אפשר (ורצוי!) לשלב ביניהם.

RESPONSIVE

עיצוב רספונסיבי

האתר צריך להיראות טוב על כל מסך - מטלפון ועד מסך 4K.

Meta Viewport

חובה ב-head כדי שהמובייל ידע לא לעשות zoom out:

head 
<meta name="viewport" content="width=device-width, initial-scale=1.0">

Media Queries

חוקי CSS שמופעלים רק בגדלי מסך מסוימים:

responsive.css 
/* עיצוב ברירת מחדל - Desktop */
.grid {
    display: grid;
    grid-template-columns: repeat(3, 1fr);
    gap: 24px;
}

/* טאבלט - מתחת ל-768px */
@media (max-width: 768px) {
    .grid {
        grid-template-columns: repeat(2, 1fr);
        gap: 16px;
    }
}

/* מובייל - מתחת ל-480px */
@media (max-width: 480px) {
    .grid {
        grid-template-columns: 1fr;  /* עמודה אחת */
    }

    .hero h1 {
        font-size: 1.5rem;          /* הקטנת כותרת */
    }
}

/* clamp - גודל רספונסיבי אוטומטי בלי media query */
h1 {
    /* מינימום | מועדף | מקסימום */
    font-size: clamp(1.5rem, 4vw, 3rem);
}

PRO TIP

CSS Variables - משתנים

הגדירו ערכים פעם אחת, השתמשו בהם בכל מקום. שינוי אחד משפיע על כל האתר.

variables.css 
/* הגדרת משתנים ב-:root (ברמה הגלובלית) */
:root {
    --primary-color: #22d3ee;
    --bg-dark: #0a0e1a;
    --text-color: #e2e8f0;
    --spacing: 16px;
    --radius: 12px;
    --font-main: 'IBM Plex Sans Hebrew', sans-serif;
}

/* שימוש במשתנים */
.card {
    background: var(--bg-dark);
    color: var(--text-color);
    padding: var(--spacing);
    border-radius: var(--radius);
    border: 1px solid var(--primary-color);
}

/* עכשיו שינוי של --primary-color
   ישפיע על כל מקום שמשתמש בו! */
💡 הדף הזה עצמו

כל הצבעים, הגופנים והמרווחים בדף הזה מוגדרים כ-CSS Variables ב-:root. פתחו את ה-DevTools (F12), שנו ערך של משתנה, ותראו את כל הדף מתעדכן בזמן אמת.

├── 📁 css/

└── 📁 pages/

└── gallery.html ← href="gallery.html" (אותה תיקיה!)

5. נתיב מוחלט (Absolute Path) - עם /

סלאש בתחילת הנתיב אומר "התחל מה-root של האתר". עובד רק על שרת (לא כקובץ מקומי):

absolute paths 
<!-- נתיב מוחלט - תמיד מה-root של האתר -->
<!-- עובד מכל דף, מכל תיקיה, בלי ../ -->
<link rel="stylesheet" href="/css/style.css">
<img src="/images/logo.png" alt="logo">
<script src="/js/app.js"></script>

<!-- URL מלא - לקבצים חיצוניים -->
<link rel="stylesheet" href="https://fonts.googleapis.com/css?family=Rubik">
<script src="https://cdn.example.com/library.js"></script>

6. טעויות נפוצות

✅ נכון
href="css/style.css"
href="../css/style.css"
src="images/logo.png"
href="about.html"
❌ שגוי
href="C:\Users\me\css\style.css"
href="css\style.css" /* backslash */
src="Images/Logo.PNG" /* case! */
href="about .html" /* space */
⚠️ שגיאות קריטיות שיישברו את הקישור

נתיב של Windows - לעולם אל תעתיקו נתיב מלא מה-Explorer של Windows (כמו C:\Users\...). זה יעבוד רק אצלכם.
Backslash - ב-HTML תמיד סלאש רגיל /, לעולם לא \.
רגישות לאותיות - בלינוקס ובשרתים, Logo.PNG זה לא logo.png. תמיד השתמשו באותיות קטנות.
רווחים בשם קובץ - הימנעו מרווחים בשמות קבצים. השתמשו ב-- או _ במקום.

7. טבלת סיכום - סוגי נתיבים

סוג דוגמה משמעות
אותה תיקיה file.css הקובץ ליד הקובץ הנוכחי
תיקיה פנימית css/file.css תיכנס לתיקיית css ותמצא שם
תיקיה למעלה ../file.css צא תיקיה אחת למעלה
שתיים למעלה ../../file.css צא שתי תיקיות למעלה
למעלה + לפנים ../css/file.css צא למעלה, היכנס ל-css
מה-root /css/file.css מהשורש של האתר (רק בשרת)
URL מלא https://... קובץ חיצוני באינטרנט
💡 טיפ פרקטי ב-VS Code

התקינו את התוסף Live Server - הוא מריץ שרת מקומי ומרענן את הדף אוטומטית כשאתם שומרים. ככה אפשר לבדוק נתיבים מוחלטים (עם /) ולראות שינויים בזמן אמת בלי לרענן ידנית.

הדרישות

קובץ 1

index.html - דף הבית

  • כותרת ראשית <h1>
  • רשימה לא ממוספרת <ul> של מוצרים
  • כל מוצר ברשימה הוא קישור לאתר חיצוני ברשת - למשל דף מוצר ב-KSP, אמזון וכו' (עם target="_blank")
  • תפריט ניווט <nav> שמקשר לשני הקבצים האחרים שלכם
קובץ 2

???.html - אתם בוחרים את השם

  • כותרת <h1>
  • טבלה <table> עם <thead> ו-<tbody>
  • 2 עמודות: "שם מוצר" ו-"מחיר"
  • תפריט ניווט <nav> לשני הקבצים האחרים
קובץ 3

???.html - אתם בוחרים את השם

  • פסקה <p> - כתבו מה שבא לכם על המוצרים
  • תמונה <img> של מוצר (הורידו תמונה מהרשת לתיקיה, או שימו URL חיצוני)
  • קישור עם תמונה - לוגו קטן שבלחיצה עליו מוביל לאתר חיצוני
  • תפריט ניווט <nav> לשני הקבצים האחרים
🔑 ההבדל החשוב

רשימת המוצרים ב-index.html מקשרת לאתרים ברשת (KSP, אמזון וכו') עם target="_blank".
תפריט הניווט (nav) מקשר בין הקבצים שלכם - בלי target="_blank" כי אלה דפים שלכם.

HTML

מבנה דף HTML

HTML - שפת תגיות שמגדירה את המבנה והתוכן של דף ווב. כל דף HTML בנוי מתגיות מקוננות שיוצרות עץ היררכי.

📄

DOCTYPE

הצהרה לדפדפן שזהו מסמך HTML5. תמיד בשורה הראשונה.

🧠

head

מטא-דאטה: כותרת הטאב, קידוד, CSS, ומידע שלא נראה בדף עצמו.

👁️

body

כל התוכן הנראה למשתמש: טקסט, תמונות, קישורים, טפסים.

השלד הבסיסי

index.html 
<!-- הצהרה שזה מסמך HTML5 -->
<!DOCTYPE html>
<html lang="he" dir="rtl">

<head>
    <!-- קידוד תווים - תמיד UTF-8 -->
    <meta charset="UTF-8">

    <!-- התאמה למובייל -->
    <meta name="viewport" content="width=device-width, initial-scale=1.0">

    <!-- כותרת שמופיעה בטאב -->
    <title>שם האתר</title>

    <!-- קישור לקובץ CSS חיצוני -->
    <link rel="stylesheet" href="style.css">
</head>

<body>
    <header>
        <nav>תפריט ניווט</nav>
    </header>

    <main>
        <h1>כותרת ראשית</h1>
        <p>תוכן הדף</p>
    </main>

    <footer>
        <p>כותרת תחתונה</p>
    </footer>
</body>

</html>
💡 טיפ

תגיות סמנטיות כמו <header>, <main>, <footer> עדיפות על <div> גנרי - הן עוזרות לקוראי מסך, ל-SEO, ולקריאות הקוד.

Box Model

כל אלמנט ב-HTML הוא קופסה. ה-Box Model מגדיר 4 שכבות סביב התוכן.

הדגמה חיה

content
box-model.css 
.element {
    /* Content - התוכן עצמו */
    width: 200px;
    height: 100px;

    /* Padding - ריווח פנימי (בין התוכן לגבול) */
    padding: 20px;            /* כל הצדדים */
    padding: 10px 20px;        /* למעלה/למטה | ימין/שמאל */
    padding: 10px 20px 30px 40px; /* top right bottom left */

    /* Border - הגבול */
    border: 2px solid #333;

    /* Margin - ריווח חיצוני (בין האלמנט לשכנים) */
    margin: 16px;

    /* חשוב! box-sizing שולט איך מחושב הרוחב */
    box-sizing: border-box;   /* width כולל padding + border */
}
🔑 קריטי

תמיד הגדירו box-sizing: border-box על כל האלמנטים (דרך ה-reset). בלי זה, padding ו-border מתווספים מעבר ל-width ויוצרים חישובים מבלבלים.

💡 מתי להשתמש במה?

Grid - כשצריך שליטה בשני ממדים: דף שלם, גלריה, דשבורד.

Media Queries

חוקי CSS שמופעלים רק בגדלי מסך מסוימים:

responsive.css 
/* עיצוב ברירת מחדל - Desktop */
.grid {
    display: grid;
    grid-template-columns: repeat(3, 1fr);
    gap: 24px;
}

/* טאבלט - מתחת ל-768px */
@media (max-width: 768px) {
    .grid {
        grid-template-columns: repeat(2, 1fr);
        gap: 16px;
    }
}

/* מובייל - מתחת ל-480px */
@media (max-width: 480px) {
    .grid {
        grid-template-columns: 1fr;  /* עמודה אחת */
    }

    .hero h1 {
        font-size: 1.5rem;          /* הקטנת כותרת */
    }
}

/* clamp - גודל רספונסיבי אוטומטי בלי media query */
h1 {
    /* מינימום | מועדף | מקסימום */
    font-size: clamp(1.5rem, 4vw, 3rem);
}

PRO TIP

CSS Variables - משתנים

הגדירו ערכים פעם אחת, השתמשו בהם בכל מקום. שינוי אחד משפיע על כל האתר.

variables.css 
/* הגדרת משתנים ב-:root (ברמה הגלובלית) */
:root {
    --primary-color: #22d3ee;
    --bg-dark: #0a0e1a;
    --text-color: #e2e8f0;
    --spacing: 16px;
    --radius: 12px;
    --font-main: 'IBM Plex Sans Hebrew', sans-serif;
}

/* שימוש במשתנים */
.card {
    background: var(--bg-dark);
    color: var(--text-color);
    padding: var(--spacing);
    border-radius: var(--radius);
    border: 1px solid var(--primary-color);
}

/* עכשיו שינוי של --primary-color
   ישפיע על כל מקום שמשתמש בו! */
💡 הדף הזה עצמו

כל הצבעים, הגופנים והמרווחים בדף הזה מוגדרים כ-CSS Variables ב-:root. פתחו את ה-DevTools (F12), שנו ערך של משתנה, ותראו את כל הדף מתעדכן בזמן אמת.

6. הגנות - איך מונעים את זה?

א. X-Frame-Options - השרת אומר לדפדפן "אל תרשה לאחרים להטמיע אותי":

HTTP Response Headers (מצד השרת) 
# אף אחד לא יכול להטמיע אותי
X-Frame-Options: DENY

# רק אני יכול להטמיע את עצמי
X-Frame-Options: SAMEORIGIN

# הגרסה המודרנית (CSP)
Content-Security-Policy: frame-ancestors 'self'
💡 נסו בעצמכם

שימו <iframe src="https://facebook.com"> בקוד שלכם ותראו שזה לא עובד. פייסבוק (ורוב האתרים הגדולים) חוסמים הטמעה עם X-Frame-Options. פתחו את ה-DevTools ← Console ותראו שגיאת "Refused to display in a frame".

ב. sandbox - "ארגז חול" שמגביל את מה שה-iframe יכול לעשות:

<!-- עם sandbox ריק: האתר לא יכול כמעט כלום -->
<!-- אנחנו נותנים הרשאות רק למה שצריך -->

<iframe
    src="https://example.com"
    sandbox="allow-scripts allow-forms"
    title="Secured Frame"
></iframe>

הרשאות sandbox שאפשר לתת:

allow-scripts הרשה הרצת JavaScript
allow-forms הרשה שליחת טפסים
allow-popups הרשה פתיחת חלונות
allow-same-origin הרשה גישה לאותו מקור
⚠️ כלל אצבע לאבטחה

תמיד בדקו שהאתר שאתם מטמיעים אמין. השתמשו ב-sandbox כשמדובר בתוכן צד שלישי. אל תטמיעו אתרים שאתם לא מכירים - הם יכולים להריץ קוד זדוני בתוך הדף שלכם.

8. טבלת סיכום - כל דרכי המיקום

צורך הפתרון דוגמה
גודל קבוע width + height על ה-iframe width="560" height="315"
רוחב מלא width="100%" או CSS width: 100%; height: 400px;
יחס 16:9 wrapper עם padding-bottom: 56.25% ראו דוגמה למעלה
יחס 16:9 (מודרני) aspect-ratio: 16/9 פשוט וקצר, אבל לא נתמך בדפדפנים ישנים
מרכוז display: block; margin: auto; או text-align: center על האב
ללא גבול border: none; בCSS עדיף על frameborder="0"
פינות מעוגלות border-radius בCSS border-radius: 12px;
💡 aspect-ratio vs padding-bottom

הטריק של padding-bottom: 56.25% עובד בכל הדפדפנים אבל מסורבל. aspect-ratio: 16/9 הוא הפתרון המודרני - פשוט, נקי, ונתמך בכל הדפדפנים החדשים (2022+). למשימת הכיתה, שניהם בסדר.

7. GET vs POST - מתי משתמשים במה?

GET - המידע ב-URL
מתי? חיפוש, פילטרים, שליפת דף

google.com/search?q=cyber+security
המידע גלוי ב-URL. מוגבל באורך. אפשר לשמור כ-bookmark.
POST - המידע חסוי ב-Body
מתי? התחברות, הרשמה, העלאת קובץ

POST /login HTTP/1.1
Body: username=admin&password=123
המידע לא גלוי ב-URL. לא מוגבל באורך. לא נשמר בהיסטוריה.
🚩 זווית סייבר - למה POST ולא GET לסיסמאות?

אם שולחים סיסמה ב-GET, היא תופיע ב-URL של הדפדפן, בהיסטוריה, בלוגים של השרת, ובכל proxy שבדרך. תמיד השתמשו ב-method="POST" לטפסי התחברות. גם POST לא מוצפן בפני עצמו - לכן חובה HTTPS.

💡 hidden inputs

שדות type="hidden" לא מוצגים למשתמש, אבל נשלחים עם הטופס. משמש לשליחת tokens, session IDs, או מידע טכני. מצד סייבר - תמיד בדקו מה נשלח בשדות מוסתרים! פתחו DevTools ← Elements ותראו אותם.

3. SQL - השפה של מסדי נתונים

SQL (Structured Query Language) היא השפה שבה מדברים עם מסד הנתונים. 4 פעולות בסיסיות:

sql-basics.sql 
-- יצירת טבלה
CREATE TABLE users (
    id       INT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50),
    password VARCHAR(255),
    email    VARCHAR(100),
    created  DATETIME DEFAULT NOW()
);

-- הוספת משתמש (כשנרשמים)
INSERT INTO users (username, password, email)
VALUES ('admin', 'hashed_pass', 'admin@site.com');

-- שליפת משתמש (כשמתחברים)
SELECT * FROM users
WHERE username = 'admin' AND password = 'hashed_pass';

-- עדכון נתונים
UPDATE users SET email = 'new@email.com'

פעולה SQL מתי?
Create INSERT INTO הרשמה, הוספת מוצר, הזמנה חדשה
Read SELECT התחברות, הצגת מוצרים, חיפוש
Update UPDATE עדכון פרופיל, שינוי סטטוס הזמנה
Delete DELETE מחיקת חשבון, הסרת מוצר

🚩 SQL Injection - המתקפה הכי נפוצה!

אם השרת שם את הקלט ישירות ב-SQL בלי הגנה, תוקף יכול להזין בשדה username: ' OR 1=1 -- ולעקוף את ההתחברות. לכן תמיד משתמשים ב-Prepared Statements (הסימני ? בדוגמה למעלה). זה נושא מרכזי בסייבר שנלמד בהמשך.

CSS חובה - השתמשו ב-<style> ב-head כדי ליצור את הגריד 3x3. בלי CSS המוצרים ייראו אחד מתחת לשני.

קישורים - כל כרטיס מוצר הוא קישור ל-info.html. הכפתור ב-info מקשר ל-order.html.
טופס - כל שדה חייב name + id + label עם for תואם.
ניווט - מכל דף אפשר להגיע לכל דף אחר. שימו nav בכל עמוד.
action="#" - שימו # ב-action כדי להישאר באותו דף (אין שרת).
